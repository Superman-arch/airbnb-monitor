FIX FOR DOCKER CONTAINER CRASH - Use LlamaFile Instead
========================================================

The Phi-4 Docker container doesn't work on Jetson Nano.
Here's the simple fix using llamafile (no Docker needed):

RUN THESE COMMANDS ON YOUR JETSON:
-----------------------------------

1. Run the setup script:
   ./setup_llamafile_jetson.sh
   
   This will:
   - Download Phi-3.5 Vision llamafile (2.5GB)
   - Create start/stop scripts
   - Update configuration

2. Start the VLM server:
   ./start_llamafile_vlm.sh
   
3. Run your monitoring system:
   python3 run_optimized.py

4. Open web interface and calibrate:
   http://<jetson-ip>:5000
   Click "Calibrate Doors"

THAT'S IT!
----------
- No Docker required
- Runs directly on Jetson
- Uses less memory than Docker
- 100% local processing

To stop the VLM server:
   ./stop_llamafile_vlm.sh

If download fails, manually download from:
https://huggingface.co/Mozilla/Phi-3.5-vision-instruct-llamafile/resolve/main/Phi-3.5-vision-instruct.Q4_K_M.llamafile
